Framework: Next.js 15 (App Router) + React 19 with TypeScript as the baseline language.
Routing Strategy: React Server Components for heavy data fetching, static generation for marketing sections, dynamic routes for product variants and category/product detail pages, route-level code splitting, and streaming with Suspense boundaries where appropriate
Deployment Target: Vercel optimized for Edge Caching, Image Optimization, ISR, and fast global delivery
2. Styling & Theming
Styling Engine: Tailwind CSS v4 with custom CSS variables for theme tokens, component-level theming, and future dark/light extensibility
Component Library: shadcn/ui + Radix Primitives + Aceternity UI
Design Tokens: Centralized tokens for color, typography, radius, shadows, spacing, motion duration, easing curves, border opacity, and semantic states such as success, warning, danger, stock urgency, and promotional highlight
3. State & Data Management
State Management: Zustand for global cart/configurator state and React Context for UI-scoped theming and presentation concerns
Data Fetching: React Server Components for initial data load, Suspense boundaries for progressive rendering, TanStack Query for client-side hydration, inventory checks, pricing refreshes, and cache-aware interactions
API Strategy: RESTful endpoints with aggressive Vercel Data Cache policies, stale-while-revalidate behavior, selective revalidation for stock and pricing, and strict schema validation at the boundary
4. Animation & Interaction
Animation Library: Framer Motion for UI and layout animation, GSAP for complex scroll-jacking and scrubbed product reveal sequences
Interaction Patterns: Scroll-linked hardware exploded views, sticky specification sidebars, hover-preview mega menus, dynamic product reveal cards, magnetic buttons, viewport-triggered stat animations, animated variant switches, and selectively applied 3D/parallax depth

📌 Part 5: Strict Engineering & Design Directives
Lean & Efficient
Minimize code; prefer composition over repetition.
Use native HTML elements whenever possible, then enhance.
Tree-shake unused library code and avoid dead weight.
Non-Generic Layouts
Reject cookie-cutter templates. Incorporate unique asymmetry, custom grids, and immersive scroll experiences.
Leverage fluid typography and container queries for dynamic scaling.
Micro-Interactions
Employ subtle yet delightful animations: hover reveals, button press feedback, section-load transitions, and tactile micro-states.
Keep animation durations and easings consistent; respect motion sensitivity preferences.
Generous Spacing & Hierarchy
Prioritize readability and brand feel with thoughtful use of white space.
Implement a clear typographic hierarchy to guide user attention.
Separation of Concerns
Isolate UI components from business logic.
Use custom hooks for data fetching and state.
Keep markup semantic and avoid side effects in components.
E-Commerce & Enterprise Logic
Account for product variations, stock availability, pricing tiers, discount logic, etc.
Provide fail-safe states for network errors and fallback content.
Accessibility (a11y)
Label every interactive element (aria-labels, roles).
Ensure keyboard navigability and focus management.
Implement color and contrast guidelines; offer motion reduction toggles.
Performance & Responsiveness
Implement responsive images and native lazy loading (e.g., Next/Image component).
Define performance budgets (bundle size, LCP, CLS).
Use code splitting at route and component level; lazy load non-critical modules.
SEO & Structured Data
Ensure semantic HTML markup for headings, lists, and links.
Include appropriate meta tags and JSON-LD schema if relevant.
Use server-side rendering or static generation to improve crawlability.
Maintainability & Scalability
Follow a clear project structure with domain-driven modules.
Use a consistent naming convention for files, components, and variables.
Document your code, particularly for complex logic and custom hooks.
